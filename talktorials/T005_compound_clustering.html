    T005 · 化合物聚类¶

    课程目标¶

    相似的化合物可能会结合到相同的靶标上并显示出类似的效果。 基于这种相似性质的原理，可以通过聚类来构建化合物的化学组。 通过这样的聚类，还可以从更大的筛选化合物集中选择出多样化的化合物组进行进一步的实验测试。

    理论¶

    Introduction to clustering and Jarvis-Patrick algorithm¶

    聚类 是 将一组对象分组的任务，使得同一组内（称为一个簇）的对象在某种意义上比其他组（簇）的对象更相似 。

    药物研究中的化合物聚类通常基于化合物之间的化学或结构相似性，以发现共享属性的组，并设计出多样化且具有代表性的集合以供进一步分析。

    一般过程如下：

    • 方法基于通过相似性对邻近点进行聚类的数据。

    • 在化学信息学中，化合物通常被编码为分子指纹，相似性可以通过Tanimoto相似度来描述（见教程T004）。

    快速提醒：

    • 指纹是二进制向量，其中每个位点表示分子内特定子结构片段的存在或缺失。

    • 相似性（或距离）矩阵：通过二进制指纹表示的分子对之间的相似性最常用Tanimoto系数来量化，该系数测量共同特征（位点）的数量。

    • Tanimoto系数的值范围从零（无相似性）到一（高相似性）。

    有多种聚类算法可供选择，其中Jarvis-Patrick聚类算法是一个在制药领域广泛使用的算法之一。 Jarvis-Patrick聚类算法由两个参数\(K\)和\(K_{min}\)定义：

    • 为每个分子计算\(K\)个最近邻的集合。

    • 如果满足以下条件，两个分子将聚类在一起：

      • 它们在彼此的最近邻列表中

      • 它们至少有 \(K_{min}\)个\(K\)个最近邻是共同的。

    Jarvis-Patrick聚类算法是确定性的，并且能够在几个小时内处理大量的分子集。然而，这种方法的一个缺点是它倾向于产生大型的异质性聚类（参见上文提到的 Butina聚类）。

    更多聚类算法可参考 scikit-learn clustering module.

    Detailed explanation of Butina clustering¶

    Butina聚类（J. Chem. Inf. Model. (1999), 39 (4), 747）是为了识别更小但同质的聚类而开发的，前提是（至少）聚类中心将比给定阈值更相似于聚类中的每个其他分子。

    以下是这种聚类方法的关键步骤（见下文流程图）：

    1. 数据准备和化合物编码¶

    • 为了识别化学相似性，输入数据中的化合物（例如，以SMILES格式给出）将被编码为分子指纹，例如RDK5指纹，这是一种基于子图的指纹，类似于著名的Daylight Fingerprint（在原始出版物中使用）。

    2. Tanimoto 相似性 (或 距离) 矩阵¶

    • 使用Tanimoto系数计算两个指纹之间的相似性。

    • 计算所有可能的分子/指纹对之间的Tanimoto相似性矩阵（\(n*n\)相似性矩阵，其中\(n\)是分子的数量，仅使用上三角矩阵）。

    • 同样，可以计算距离矩阵（\(1 - 相似性\)）。

    3. 聚类分子：质心和排除球¶

    注意：如果分子与聚类质心的最大距离低于指定的截止值（如果使用距离矩阵），或者它们具有高于指定截止值的最小相似性（如果使用相似性矩阵），则分子将被聚类在一起。

    • 潜在聚类质心的识别

      • 聚类质心是给定聚类中具有最多邻居的分子。

      • 注释邻居：对于每个分子，计算所有Tanimoto距离低于给定阈值的分子数量。

      • 按分子的邻居数量降序排序，以便潜在的聚类质心（即具有最多邻居的化合物）位于文件的顶部。

    • 基于排除球的聚类

      • 从排序列表中的第一个分子（质心）开始。

        • 所有Tanimoto指数高于或等于聚类截止值的分子随后成为该聚类的成员（在相似性的情况下）。

          • 被确定为给定聚类成员的每个分子都会被标记并从进一步比较中移除。因此，被标记的分子不能成为另一个聚类质心或另一个聚类的成员。这个过程就像是在新形成的聚类周围放置了一个排除球。

          • 一旦列表中的第一个化合物找到了所有它的邻居，列表中第一个可用的（即未被标记的）化合物就成为新的聚类质心。

        • 对列表中所有其他未标记的分子重复相同的过程。

      • 在聚类过程结束时未被标记的分子成为单例。

        • 注意，一些被指定为单例的分子在给定的Tanimoto相似性指数下可能有邻居，但这些邻居已经被更强的聚类质心排除了。

    挑选多样化的化合物¶

    在制药行业中，寻找代表性化合物集合是一个常用概念。

    • 假设我们进行了一次虚拟筛选活动，但只有有限的资源在确认性实验中测试少数化合物。

    • 为了从这次筛选中获得尽可能多的信息，我们希望选择一个多样化的集合。因此，我们在潜在活性化合物的列表中挑选每个化学系列的一个代表。

    另一种情况可能是选择一个系列来获取有关结构-活性关系的信息；即，分子中的小结构变化如何影响 体外 活性。

    Tanimoto 相似度和距离矩阵¶

    现在我们生成了指纹，我们继续下一步：识别潜在的集群质心。 为此，我们定义了计算 Tanimoto 相似度和距离矩阵的函数。

    [ ]:

    def tanimoto_distance_matrix(fp_list):
    """计算指纹列表的距离矩阵"""
    dissimilarity_matrix = []
    # 请注意，我们是如何故意跳过列表中的第一项和最后一项的，因为我们不需要将它们与自身进行比较
    for i in range(1, len(fp_list)):
        # 将当前指纹与列表中所有先前的指纹进行比较
        similarities = DataStructs.BulkTanimotoSimilarity(fp_list[i], fp_list[:i])
        # 由于我们需要一个距离矩阵，因此为相似性矩阵中的每个元素计算 1-x
        dissimilarity_matrix.extend([1 - x for x in similarities])
    return dissimilarity_matrix

    聚集分子：质心和排除球体¶

    在这部分，我们将分子聚类并查看结果。

    定义聚类函数。

    [ ]:

    def cluster_fingerprints(fingerprints, cutoff=0.2):
    """Cluster fingerprints
    Parameters:
        fingerprints
        cutoff: threshold for the clustering
    """
    # 计算 Tanimoto 距离矩阵
    distance_matrix = tanimoto_distance_matrix(fingerprints)
    # 现在使用实现的 Butina 算法对数据进行聚类：
    clusters = Butina.ClusterData(distance_matrix, len(fingerprints), cutoff, isDistData=True)
    clusters = sorted(clusters, key=len, reverse=True)
    return clusters

    如何选择合理的截止值？¶

    由于聚类结果取决于用户选择的阈值，因此我们将仔细研究截止值的选择。

    [14]:

    clusters = cluster_fingerprints(fingerprints, cutoff=cutoff)
    fig, ax = plt.subplots(figsize=(15, 4))
    ax.set_title(f"Threshold: {cutoff:3.1f}")
    ax.set_xlabel("Cluster index")

    display(fig)

    如您所见，阈值（距离截止）越高，被认为相似的分子就越多，因此聚集成较少的簇。 阈值越低，出现的小集群和 “单例” 就越多。

    距离值截断值越小，化合物属于一个簇所需的相似度就越高。

    查看上面的图表，我们决定选择0.2.单例不多，集群大小没有极端但平滑的分布。

    集群内 Tanimoto 的相似之处¶

    我们还可以看看集群内 Tanimoto 的相似性。

    [ ]:

    def intra_tanimoto(fps_clusters):
    """计算每个集群中所有指纹对的 Tanimoto 相似性的函数"""
    intra_similarity = []
    # 计算每个聚类的内部相似性
    for cluster in fps_clusters:
        # 转换为相似性矩阵（1 距离）的谷本距离矩阵函数
        intra_similarity.append([1 - x for x in tanimoto_distance_matrix(cluster)])
    return intra_similarity

    选择最终的化合物列表¶

    在下文中，我们将选择最终的 最多 1000 种化合物 作为 不同 子集。

    为此，我们从每个簇（即每个簇的第一个分子）中取出簇质心，然后对于每个簇（从最大的一个开始），我们取与质心最相似的 10 个分子（如果簇中剩余的分子少于 10 个，则为 50%），直到我们选择了最多 1000 个化合物。因此，我们有每个集群的代表。

    这种化合物选择的目的是确保提议在验证性分析中测试的较小化合物集的多样性。

    拣选程序改编自S. Riniker 和 G. Landrum 编写的 TDT 教程.

    正如那里所描述的：这种方法背后的想法是确保多样性（每个簇的代表），同时从验证性分析的结果中获得一些 SAR（构效关系）（保留了来自较大簇的非常相似的分子组）。

    获取集群中心。

    [ ]:

    # 获取每个簇的簇中心（每个簇中的第一个分子）
cluster_centers = [compounds[c[0]] for c in clusters]
# 我们有多少个集群中心/集群？
print("Number of cluster centers:", len(cluster_centers))
# NBVAL_CHECK_OUTPUT

    Number of cluster centers: 1128

    按大小对簇进行排序，按相似性对每个簇中的分子进行排序。

    [ ]:

    # 根据分子与簇中心的相似性对簇中的分子进行排序，并根据簇的大小对簇进行排序
sorted_clusters = []
for cluster in clusters:
    if len(cluster) <= 1:
        continue  # singletons else （其他单例）：
    # 计算每个群集元素的指纹
    sorted_fingerprints = [rdkit_gen.GetFingerprint(compounds[i][0]) for i in cluster]
    # 所有集群成员与集群中心的相似性
    similarities = DataStructs.BulkTanimotoSimilarity(
        sorted_fingerprints[0], sorted_fingerprints[1:]
    )
    # 将分子的索引添加到其相似性中（不包括质心！）
    similarities = list(zip(similarities, cluster[1:]))
    # 按相似度降序排序
    similarities.sort(reverse=True)
    # 以 clusters_sort 保存分子的簇大小和索引
    sorted_clusters.append((len(similarities), [i for _, i in similarities]))
    # 按集群大小降序排序
    sorted_clusters.sort(reverse=True)

    最多选择 1000 个化合物。

    [ ]:

    # 对选定的分子进行计数，首先选择簇中心
selected_molecules = cluster_centers.copy()
# 从最大的一个开始，从每个簇中取出 10 个分子（或最多 50%）。
index = 0
pending = 1000 - len(selected_molecules)
while pending > 0 and index < len(sorted_clusters):
    # 获取已排序集群的索引
    tmp_cluster = sorted_clusters[index][1]
    # If the first cluster is > 10 big then take exactly 10 compounds
    if sorted_clusters[index][0] > 10:
        num_compounds = 10
    # 如果较小，则取一半的分子
    else:
        num_compounds = int(0.5 * len(tmp_cluster)) + 1
    if num_compounds > pending:
        num_compounds = pending
    # 将拾取的分子及其结构写入名为 picked_fps 的列表列表中
    selected_molecules += [compounds[i] for i in tmp_cluster[:num_compounds]]
    index += 1
    pending = 1000 - len(selected_molecules)
print("# Selected molecules:", len(selected_molecules))
